# Feature Specification: Fix Shot Physics

**Feature Branch**: `003-fix-shot-physics`  
**Created**: 2026-04-20  
**Status**: Draft  
**Input**: User description: "修复当前版本的击球物理：1）首次击球/开球速度过慢；2）白球击中球堆之后力度过大（疑似碰撞增能）。目标：碰撞不再注入能量，开球速度标定合理，补充必要单测/回归测试。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 开球速度与距离符合直觉 (Priority: P1)

作为玩家，我希望首次击球（开球）在相同力度输入下，白球移动速度与距离符合直觉，不会“很慢拖很久”，并且能稳定碰到球堆。

**Why this priority**: 开球是最早触达的核心交互，过慢会让玩家误判玩法与物理可信度。

**Independent Test**: 在默认桌面与默认摆球布局下，仅通过一次满力度开球即可验证“能在合理时间内发生首碰”。

**Acceptance Scenarios**:

1. **Given** 默认摆球与默认开球方向，**When** 玩家以满力度开球，**Then** 白球应在可接受时间内与球堆发生首次碰撞。
2. **Given** 默认摆球与默认开球方向，**When** 玩家以中等力度开球，**Then** 白球仍应稳定触发首次碰撞（不需要极端输入才碰到球堆）。

---

### User Story 2 - 撞击球堆后不出现异常爆能 (Priority: P1)

作为玩家，我希望白球击中球堆后，球堆散开强度与输入力度相匹配，不出现“撞到球堆后力度突然过大/越撞越快”的异常行为。

**Why this priority**: 球-球碰撞是台球玩法的核心。若碰撞会异常增能，会导致不可预测的爆散，破坏体验与规则判断。

**Independent Test**: 构造一个两球碰撞的最小回归场景，验证碰撞不会导致“总运动强度”异常放大。

**Acceptance Scenarios**:

1. **Given** 两球发生一次球-球碰撞的最小场景，**When** 碰撞发生，**Then** 碰撞后两球速度变化应符合直觉（不会出现越撞越快）。
2. **Given** 白球开球击中球堆，**When** 发生多次连锁碰撞，**Then** 球堆散开强度与开球力度相匹配，不出现明显异常爆能。

---

### User Story 3 - 修复不引入规则与判定回归 (Priority: P2)

作为开发/测试，我希望在修复物理后，首碰检测、停球判定、进袋判定等现有行为仍按预期工作。

**Why this priority**: 物理修复必须是可控且可回归的，不应破坏已有对局闭环。

**Independent Test**: 运行现有测试 + 新增回归测试即可验证。

**Acceptance Scenarios**:

1. **Given** 现有首碰检测场景，**When** 运行回归测试，**Then** 仍能检测到首次碰撞。
2. **Given** 速度低于停止阈值的球，**When** 物理步进执行，**Then** 球会被判定停止。

### Edge Cases

- 小角度掠碰不应出现反常“加速”或方向翻转。
- 多球短时间内连续碰撞不应导致速度逐次放大。
- 极小力度输入不应触发异常运动或数值爆炸。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 修复球-球碰撞导致的异常增能现象，使碰撞后的运动强度与输入力度一致、可预测。
- **FR-002**: 系统 MUST 调整开球（首次击球）速度标定，使默认布局下开球不再显著“过慢”。
- **FR-003**: 系统 MUST 提供自动化回归测试覆盖“碰撞不增能”和“开球可在合理时间内首碰”。
- **FR-004**: 系统 MUST 保持首碰检测、停球判定、进袋判定等现有行为不回归。

### Key Entities *(include if feature involves data)*

- **Ball**: 一颗球的状态（位置、速度、半径、质量、是否进袋）。
- **Physics Config**: 物理参数集合（步长、衰减、恢复系数、出杆速度上限等）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在默认摆球与默认开球方向下，满力度开球可在 1.5 秒内触发首次碰撞（可通过自动化测试验证）。
- **SC-002**: 在两球碰撞的回归测试中，碰撞后“总运动强度”不应显著高于碰撞前（允许 ≤2% 的数值误差区间）。
- **SC-003**: `npm test`、`npm run lint`、`npm run build` 全部通过。

## Assumptions

- 桌面坐标尺度与时间步进保持不变；本次只修复物理行为与标定，不更改玩法规则。
- 玩家力度输入仍以现有方式映射为 0..1 的力度百分比。
- 允许在不改变对局规则的前提下微调物理参数以达到合理手感。
