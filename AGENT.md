# AGENT 代码编写与提交注意事项

本文件用于记录本项目在开发、测试、提交与协作流程中的约束与习惯做法，供后续开发与自动化 Agent 执行时遵循。

## 提交与协作规范

- `git commit` 信息与 MR（合并请求）标题/描述 **统一使用中文**编写。

## 变更记录规范（必须）

- 每次关键改动与提交代码前后，都需要在以下多维表格中留下**文字描述**与**截图记录**：
  - `https://bytedance.larkoffice.com/base/FbKAbZJHFa60ZcsLj1vcIpIqn6g`

建议记录内容包含：

- 改动目标与范围
- 关键实现点/风险点
- 验证方式（截图/日志/测试用例）

## 提交前质量门禁（必须）

- 提交代码之前，必须确保：
  - `lint` 全部通过
  - 单元测试（`test`）全部通过
